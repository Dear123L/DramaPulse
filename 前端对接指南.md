# DramaPulse 前端对接指南

> 本文档面向 **Android 前端开发者**，只列出你在 App 里真正需要调用的接口。
> 后端服务启动后基地址为 `http://<服务器IP>:8000`

---

## 快速概览

前端 App 的核心交互流程：

```
用户打开App → 加载剧集信息 → 播放视频 → 到达高光点 → 触发特效 → 弹出分支选择 → AI续写/配图
```

对应的接口调用链：

```
GET /episodes/67/highlights    →  获取所有高光点（含触发时机、特效配置、分支选项）
POST /ai/branch-with-image     →  用户选分支后调用，返回AI续写文字+配图
GET /video/stream              →  播放主视频
```

---

## 接口列表

### 1. 获取高光点列表（核心接口）

**前端在什么时候调用：** 用户进入某集播放页时调用一次，拿到所有高光点配置，在播放器中按时间戳注册触发器。

| 项 | 值 |
|---|---|
| **方法** | `GET` |
| **路径** | `/episodes/{episode_no}/highlights` |
| **所在文件** | `routes/episodes.py` |
| **示例** | `GET /episodes/67/highlights` |

**响应示例：**

```json
{
  "episode_no": 67,
  "title": "古墓谜影·第六十七集",
  "theme": "古墓探险",
  "total_duration": 2700,
  "highlight_count": 5,
  "highlights": [
    {
      "id": 9,
      "timestamp": 450,
      "type": "suspense",
      "intensity": 8,
      "emotion": "紧张",
      "scene_desc": "黑暗中传来异响，众人屏息",
      "action_desc": "角色缓缓举起火把",
      "trigger": {
        "type": "auto",
        "delay_ms": 0
      },
      "visual_effect": "screen_shake",
      "particle_type": "dust",
      "haptic_pattern": "heavy_pulse",
      "audio_cue": "drum_hit",
      "show_branch": true,
      "branch_options": [
        { "id": "A", "text": "探查左侧暗道" },
        { "id": "B", "text": "原路返回" },
        { "id": "C", "text": "在岔口做标记" }
      ],
      "ai_prompt": ""
    }
  ]
}
```

**前端如何使用各字段：**

| 字段 | 用途 |
|---|---|
| `timestamp` | 视频播放到这个秒数时触发特效（单位：秒） |
| `intensity` | 特效强度 1-10，前端可据此调节震动/粒子数量 |
| `visual_effect` | 视觉特效类型：`screen_shake`(屏幕抖动)、`flash`(闪光)、`vignette`(暗角)等 |
| `particle_type` | 粒子特效：`dust`(灰尘)、`spark`(火花)、`mist`(迷雾)等 |
| `haptic_pattern` | 触觉反馈模式：`heavy_pulse`(重脉冲)、`light_tap`(轻敲)等 |
| `audio_cue` | 音效标识：`drum_hit`(鼓点)、`whisper`(低语)等 |
| `show_branch` | 是否弹出分支选择弹窗 |
| `branch_options` | 分支选项列表，`id` 用于后续调AI接口，`text` 显示给用户 |

### 2. 用户选择分支 — AI续写+配图（核心接口）

**前端在什么时候调用：** 用户在分支弹窗中选择某个选项后调用。

| 项 | 值 |
|---|---|
| **方法** | `POST` |
| **路径** | `/ai/branch-with-image` |
| **所在文件** | `routes/ai_branch_image.py` |
| **降级链** | 先调此接口；如果返回的 `image_url` 为空，则只展示文字 |

**请求体：**

```json
{
  "episode_no": 67,
  "highlight_id": 9,
  "branch_id": "A",
  "branch_text": "探查左侧暗道",
  "context": ""
}
```

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `episode_no` | int | 否 | 集数，默认67 |
| `highlight_id` | int | 是 | 高光点ID，从接口1获取 |
| `branch_id` | str | 是 | 选项ID：A/B/C，从接口1的 `branch_options` 获取 |
| `branch_text` | str | 否 | 选项文字，不传则后端从数据库自动取 |
| `context` | str | 否 | 额外上下文，一般留空 |

**响应示例（有图片）：**

```json
{
  "branch_id": "A",
  "text": "你举起火把，缓缓走向左侧的暗道。火光映照下，石壁上的古文逐渐显现...",
  "image_url": "/assets/ep67/hl9_branchA_1718000000.png",
  "result_type": "image",
  "cached": true,
  "token_usage": 350
}
```

**响应示例（无图片，纯文字）：**

```json
{
  "branch_id": "A",
  "text": "你举起火把，缓缓走向左侧的暗道...",
  "image_url": "",
  "result_type": "image",
  "cached": true,
  "token_usage": 350
}
```

**前端如何使用：**

- `text` → 显示在分支结果弹窗中
- `image_url` → 拼接基地址加载图片：`http://<服务器IP>:8000` + `image_url`
- 若 `image_url` 为空，只展示文字即可

### 3. 获取主视频流

**前端在什么时候调用：** 用户进入播放页时，作为 ExoPlayer 的视频源。

| 项 | 值 |
|---|---|
| **方法** | `GET` |
| **路径** | `/video/stream` |
| **所在文件** | `main.py` |
| **示例** | `GET /video/stream` |

**返回：** `video/mp4` 文件流，直接作为 ExoPlayer 的 MediaItem URI。

```
http://<服务器IP>:8000/video/stream
```

### 7. 纯文字AI续写（降级方案）

**前端在什么时候调用：** 这是接口2的纯文字版本，不生成配图。一般不直接调用，仅在接口2不需要图片时才考虑。

| 项 | 值 |
|---|---|
| **方法** | `POST` |
| **路径** | `/ai/branch` |
| **所在文件** | `routes/ai_branch.py` |

**请求体：** 与接口2相同（去掉 `branch_text` 和 `context` 也可）

**响应示例：**

```json
{
  "branch_id": "A",
  "text": "你举起火把，缓缓走向左侧的暗道...",
  "cached": true,
  "token_usage": 350
}
```

> **注意：** 此接口只返回 `text`，无 `image_url`。推荐优先使用接口2（`/ai/branch-with-image`）。

## 静态资源访问

后端通过 FastAPI StaticFiles 挂载了两个静态目录：

| URL前缀 | 对应磁盘目录 | 用途 |
|---|---|---|
| `/assets/` | `backend/multimodal_assets/` | AI生成的图片 |
| `/static/` | `backend/frontend/` | 测试用前端静态文件 |

**前端访问图片示例：**

数据库中存的 `image_url` 是 `/assets/ep67/hl9_branchA_xxx.png`，前端拼接基地址即可：

```
http://<服务器IP>:8000/assets/ep67/hl9_branchA_xxx.png
```

用 Glide/Coil 加载：

```kotlin
// Glide
Glide.with(context)
    .load("http://<服务器IP>:8000" + branchResult.imageUrl)
    .into(imageView)

// Coil
AsyncImage(
    model = "http://<服务器IP>:8000${branchResult.imageUrl}",
    contentDescription = "分支场景图"
)
```

## 前端对接流程图

```
App启动
  │
  ▼
GET /episodes/67/highlights     ← 获取全部高光点
  │
  ▼
启动ExoPlayer播放视频
GET /video/stream               ← 视频源
  │
  ▼
播放器监听进度 → 到达timestamp秒数
  │
  ├── 触发visual_effect（屏幕抖动/闪光/暗角）
  ├── 触发particle_type（灰尘/火花/迷雾粒子）
  ├── 触发haptic_pattern（手机震动）
  ├── 触发audio_cue（音效）
  │
  └── show_branch == true ?
       │
       ├── YES → 弹出分支选择弹窗，显示branch_options
       │         │
       │         ▼ 用户点击某个选项
       │   POST /ai/branch-with-image
       │         │
       │         ▼
       │   显示AI续写文字(text) + 配图(image_url)
       │
       └── NO  → 继续播放
```

## 不需要前端调用的接口

以下接口是后台管理/分析用的，前端不需要对接：

| 路径 | 方法 | 说明 |
|---|---|---|
| `/analyze` | POST | 触发视频分析（后端离线操作） |
| `/analyze/async` | POST | 异步触发视频分析 |
| `/ai/branch-video` | POST | 预渲染视频分支（暂无视频文件，调用必404） |
| `/ai/branch-video/{ep}/{hl}/{br}` | GET | 直接返回视频文件（同上） |
| `/ai/branch-video/list` | GET | 列出预渲染视频（调试用） |
| `/player` | GET | 测试用前端页面 |
| `/api/summary` | GET | API摘要（调试用） |
| `/docs` | GET | Swagger文档页面 |

## 数据库表结构参考

前端不需要直接访问数据库，但了解表结构有助于理解接口返回值：

### episodes 表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int | 主键 |
| episode_no | int | 集数编号 |
| title | varchar | 剧集标题 |
| theme | varchar | 主题 |
| total_duration | int | 总时长（秒） |
| status | varchar | 状态：pending/analyzing/done/failed |

### highlights 表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int | 主键（即接口中的 highlight_id） |
| episode_id | int | 所属剧集 |
| timestamp | int | 触发时间点（秒） |
| highlight_type | varchar | 高光类型 |
| emotion | varchar | 情绪标签 |
| intensity | int | 强度1-10 |
| visual_effect | varchar | 视觉特效标识 |
| particle_type | varchar | 粒子特效标识 |
| haptic_pattern | varchar | 触觉反馈标识 |
| audio_cue | varchar | 音效标识 |
| show_branch | tinyint | 是否显示分支弹窗 |
| branch_options | json | 分支选项列表 |

### branch_results 表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int | 主键 |
| highlight_id | int | 关联高光点 |
| branch_id | varchar | 分支标识 A/B/C |
| branch_text | varchar | 选项文字 |
| ai_response | text | AI续写结果（即接口返回的 text） |
| media_path | varchar | 图片URL路径（如 /assets/ep67/xxx.png） |
| result_type | varchar | 结果类型：text/image/video |
| token_usage | int | AI消耗token数 |

## 环境配置

前端开发者拉取 `backend/` 后：

1. 安装 Python 依赖：`pip install -r requirements.txt`
2. 配置 `.env` 文件（参考 `.env.example`）
3. 建数据库和表：`python run_create_tables.py`
4. 导入高光点数据：`python import_json_to_mysql.py`
5. 启动服务：`python main.py`
6. 验证：浏览器打开 `http://localhost:8000/docs` 看Swagger文档
